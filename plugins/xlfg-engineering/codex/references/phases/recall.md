# Recall Phase

## Objective

Perform deterministic recall first, before broad repo fan-out, and write the smallest relevant carry-forward memory.

## Required Artifact

Update `memory-recall.md`.

## Process

- Read `docs/xlfg/knowledge/current-state.md` first when present.
- Search durable knowledge with targeted lexical queries before scanning source broadly.
- Prefer exact file paths, run ids, and cited prior lessons over vague memory.
- Record either strong hits or an explicit no-hit result.
- Carry forward only context that changes this run's decisions, proof, or risk.

## Done Check

`memory-recall.md` exists, names the recall sources checked, and either cites relevant prior facts or records that no relevant durable memory was found.
